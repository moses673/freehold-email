# Self-Hosting Freehold Email

Freehold Email is yours. You bought it, you own it, you run it on your own server. No monthly fees. No kill-switch.

This guide gets you from zero to sending emails in about 10 minutes.

## What You Need

- A server with Docker installed (any $5-10/month VPS works: DigitalOcean, Hetzner, Linode, Railway, Fly.io)
- A domain name (for sending emails)
- A free Postmark account (https://postmarkapp.com — handles email delivery)

## Quick Start

```bash
# 1. Clone the repo (or download and extract the ZIP)
git clone https://github.com/your-username/freehold-email.git
cd freehold-email

# 2. Run setup
./setup.sh

# 3. Edit .env with your Postmark API key and domain
nano .env

# 4. Start the app
docker compose up -d
```

Your app is now running at `http://your-server-ip:3001`.

## Step-by-Step

### 1. Get a Server

Any Linux VPS with Docker works. Recommended:

| Provider | Monthly Cost | Notes |
|----------|-------------|-------|
| DigitalOcean | $6/mo | 1GB RAM, easiest setup |
| Hetzner | $4/mo | 2GB RAM, best value |
| Linode | $5/mo | 1GB RAM, reliable |
| Railway | $5/mo | No Docker needed (use railway.json) |
| Fly.io | $5/mo | Edge deployment |

Install Docker on your server:
```bash
curl -fsSL https://get.docker.com | sh
```

### 2. Get a Postmark Account

Postmark handles the actual email delivery. You pay Postmark directly for sending — Freehold has no say in this and takes no cut.

1. Sign up at https://postmarkapp.com
2. Verify your sending domain (add DNS records they give you)
3. Copy your **Server API Token** from the server's API Tokens page

Postmark's free tier sends 100 emails/month. Paid plans start at $15/month for 10,000 sends.

### 3. Configure Environment

```bash
cp .env.example .env
nano .env
```

Fill in:
- `POSTMARK_API_KEY` — your Postmark Server API Token
- `POSTMARK_FROM_EMAIL` — the verified email address you'll send from
- `APP_URL` — your server's public URL (e.g., `https://mail.yourdomain.com`)
- `JWT_SECRET` — auto-generated by setup.sh, or run `openssl rand -hex 32`
- `UNSUBSCRIBE_SECRET` — auto-generated by setup.sh, or run `openssl rand -hex 32`

### 4. Start the App

```bash
docker compose up -d --build
```

Check it's running:
```bash
docker compose logs -f
# Look for: "Freehold Email Server — Port: 3001"
```

Visit `http://your-server-ip:3001` in your browser.

### 5. Set Up HTTPS (Recommended)

For production, put nginx or Caddy in front as a reverse proxy with SSL.

**With Caddy (easiest):**
```bash
# Install Caddy
sudo apt install -y caddy

# Edit Caddy config
sudo nano /etc/caddy/Caddyfile
```

```
mail.yourdomain.com {
    reverse_proxy localhost:3001
}
```

```bash
sudo systemctl restart caddy
```

Caddy auto-provisions HTTPS certificates. Your app is now at `https://mail.yourdomain.com`.

Update `APP_URL` and `CLIENT_URL` in `.env` to use the HTTPS URL, then restart:
```bash
docker compose restart
```

## Backups

Your database is stored in a Docker volume. Back it up:

```bash
# Create backup
docker run --rm \
  -v freehold-email-data:/data \
  -v $(pwd):/backup \
  alpine tar czf /backup/freehold-backup-$(date +%Y%m%d).tar.gz /data

# Restore from backup
docker compose down
docker run --rm \
  -v freehold-email-data:/data \
  -v $(pwd):/backup \
  alpine sh -c "cd / && tar xzf /backup/freehold-backup-YYYYMMDD.tar.gz"
docker compose up -d
```

## Updating

When a new version is available:

```bash
git pull
docker compose up -d --build
```

Your data is preserved in the Docker volume — updates never touch your database.

## Troubleshooting

**App won't start:**
```bash
docker compose logs -f
# Check for missing env vars or port conflicts
```

**Emails not sending:**
- Check your Postmark API key is correct
- Verify your sending domain in Postmark (DKIM + Return-Path)
- Make sure Postmark account is approved (not in Test mode)

**Database issues:**
```bash
# Reset database (WARNING: deletes all data)
docker compose down
docker volume rm freehold-email-data
docker compose up -d --build
```

**Port conflict:**
```bash
# Change the host port in docker-compose.yml or .env
PORT=8080 docker compose up -d
```
